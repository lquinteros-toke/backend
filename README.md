# Welcome

This documentation centralizes the technical structure of the BudHub, Toke, and KushPoint ecosystem.\
Its purpose is to define how data is organized, how each app interacts with the others, and which system owns each part of the data.



### **Purpose of this documentation**

* Provide a single source of truth for data models, rules, and integrations.
* Align developers, product, and design on how each app works.
* Define ownership, structure, and behavior of all core entities.
* Prevent inconsistencies across the three applications.
